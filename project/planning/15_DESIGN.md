## §0 — Visual Direction

> **SSOT:** [14_DESIGN_BRIEF.md](./14_DESIGN_BRIEF.md). Esta sección resume; ese archivo manda en conflictos.

| Dimensión           | Decisión           | Notas                                                            |
| ------------------- | ------------------ | ---------------------------------------------------------------- |
| **Skin family**     | `editorial`        | Book-meets-product. Serif headlines + sans body. Warm neutrals.  |
| **Whitelabel mode** | `client`           | Tokens son la identidad. Assets (logo, naming final) pendientes. |
| **Theme swap**      | `runtime`          | CSS variables, no buildtime constants.                           |
| **Profile**         | `saas-app`         | App de uso diario, no marketing-hero ni dashboard.               |
| **Variance**        | medium             | Personalidad sin caer en literal-libro.                          |
| **Motion**          | calm               | 120-320ms, ease-standard. Sin spring/bounce.                     |
| **Density**         | medium             | Spacious mobile (16-24px), comfortable desktop.                  |
| **Platforms**       | web + mobile (PWA) | Mobile-first; desktop responsive.                                |
| **Dark mode**       | optional toggle    | Light canonical default (resolves OQ-3).                         |
| **Accessibility**   | WCAG AA            | Validated via axe-core en CI.                                    |
| **Locale**          | multi-lang         | es-MX default + en.                                              |

**Color palette (resumen — ver Brief §6 para OKLCH completo):**

- Background: cream `#FBF7EF` · Card: `#F2EDDF` · Sunken: `#E8E2D2`
- Ink: warm charcoal `#2A2826` · Soft: `#4A4540` · Hint: `#7A6E64`
- Rule (dividers): warm ecru `#C9BAA5` (NO gray)
- Primary CTA: warm charcoal bg + cream text (NO blue, NO purple)
- Per-scope accents (secondary signal only): Day=#5C5C5C, Week=#1F1F1F, Quarter=sage, Year=burnt orange, 5Year=steel blue, Life=wine red

**Typography:**

- Display: Source Serif 4 / Lora — weight 500
- Body: Inter — weight 400-500
- Italic serif para reflective prompts del agente

**HARD BANS** (de Brief §2 anti-patterns):

- ❌ Purple/violet como primary (LILA BAN)
- ❌ Azul SaaS genérico (#2563EB, #6366F1)
- ❌ Glassmorphism / blur exagerado
- ❌ Neumorphism / sombras dobles
- ❌ Bouncy spring transitions
- ❌ Gradientes saturados
- ❌ Emojis decorativos en copy (solo funcionales: 🎙️🔥⊙🌱🤍)
- ❌ Coachy microcopy ("¡Tú puedes!")
- ❌ All-caps buttons
- ❌ Gamification visible (deferred v2)
- ❌ Photography stock, ilustraciones cute, mascotas

**Trigger:** User en SCR-001 Landing → tap "Empezar"
**Linked US:** US-001, US-005

```

  ↓ tap "Continuar con Google"

SCR-016 Onboarding · Google Calendar opt-in

  ↓ tap "Listo"
SCR-020 Today (empty state)
```

**Side effects:** create User + NotificationPref + Category 'Inbox' + Project 'Inbox' + Subscription 'free'. Schedule recurring Inngest events (morning/evening/weekly).

### FLW-003 — Captura por voz

**Linked US:** US-070..073
**Trigger:** Tap mic FAB en SCR-020 Today (o en cualquier screen con FAB visible)

```
SCR-020 Today
  ↓ tap mic FAB (56×56px bottom-right)
  ↓ [first time] browser native mic permission prompt
SCR-050 Voice capture sheet (bottom-sheet mobile / centered modal desktop)
  ├ Waveform animado mientras graba
  ├ Transcript streaming text visible
  └ "🎙️ escuchando..." status indicator
  ↓ user taps "Listo" or 2s silence detected
  ↓ POST /api/ai/parse-task with text
SCR-050 evolves into Preview state
  ├ title (editable)
  ├ project (best guess + alternatives)
  ├ scheduled_date / time
  ├ priority (if mentioned)
  ├ deadline (if mentioned)
  ├ [Confirmar] [Editar más] [Cancelar]
  ↓ tap Confirmar
[ Server Action createActivity ]
  ↓ success
SCR-020 Today refreshed (activity visible)
  + Toast "Guardado."
```

**Fallback path:** if Web Speech API unavailable → record blob → upload to `/api/voice/transcribe` → Whisper → continues at parse-task.

**Failure paths:**

- Mic denied → CTA "Crear con teclado" → SCR-051 Activity quick-add
- Whisper API 502 → Toast "Algo se rompió. Intentá de nuevo." + retain transcript en clipboard
- Parse-task returns empty title → Modal "No entendí. ¿Repetís o lo tipeás?"

### FLW-004 — Morning ritual (check-in matutino)

**Linked US:** US-080, US-030
**Trigger:** Inngest event `morning.check_in.due` at user's `morning_time`

```
[ Background: ProactiveTask created + Web Push sent ]
  ↓ user taps push
SCR-023 Chat (deep link with context=morning_check)
  ├ Agent message 1: "Buenos días. ¿Cuál es la intención de hoy — una sola frase?"
  ↓ user types or speaks reply
  ↓ [if vague] Vague-language challenge fires
  ├ Agent message 2: "¿Qué significa eso concretamente?"
  ↓ user clarifies
  ├ Agent message 3: "¿Por qué estás agradecido hoy?"
  ↓ user replies
  ├ Agent message 4: "Hoy soy alguien que..." (identity_statement)
  ↓ ...continues through: 3 wins → avoidance → energy (3 sliders)
  ├ Final: "Guardado. Te busco al mediodía."
[ Side: DaySheet row populated, morning_completed_at = now ]
  ↓ user closes chat
SCR-020 Today (now with DaySheet morning visible)
```

**Branches:**

- User abandons midway → can resume same day; partial fields preserved
- User in `listening` intensity → no challenges fire
- User says "no quiero hacer esto hoy" → Agent: "Está bien. ¿Salgo o nos vemos al mediodía?" → if exit, DaySheet stays empty

### FLW-005b — Close-day modal (reemplaza FLW-005)

**Linked US:** US-031b
**Trigger:** Push at `evening_time` → tap notification → SCR-060 Close-day modal abre encima de SCR-020 Today

```
SCR-060 Close-day modal
  ├ Title: "Cerrar [día] — [N] actividades"
  ├ Per-activity row (cada Activity con scheduled_dates del día):
  │   ├ [✓] Hecha    → status = 'done', cerrada-checkbox opcional
  │   ├ [~] Avanzada → status = 'in_progress', slider 0-100 → progress_percent
  │   └ [ ] No la toqué → status sin cambio o 'skipped' (user elige razón opcional)
  ├ One-line input: "Una frase del día" → DaySheet.close_summary
  └ [Cerrar el día]
[ DaySheet.close_summary populated, close_completed_at = now ]
[ Each Activity status/progress_percent updated ]
  ↓ Optional: AI generates resumen agregado vía FT-104 (no edita actividades — solo nota privada)
  ↓ Volver a SCR-020 con banner "Día cerrado · [hora]"
```

**Por qué cambió:** el flujo conversacional pedía 4-5 turnos para cerrar el día. Con close-day modal, el user cierra todas sus activities + escribe una frase en ~20s. Wins ya se llenaron en morning (FLW-004), no se repiten.

> ⚠️ Iteración prototipo: nav es **SIEMPRE horizontal abajo** en todos los breakpoints (mobile / tablet / desktop). El desktop sidebar original fue eliminado. 7 items totales. Ver `src/components/agenda/AgendaBottomNav.tsx`.

### All breakpoints — Bottom nav horizontal SIEMPRE

```
┌─────────────────────────────────┐
│         App Header              │  ← 56px height; transparent on Today, bg-elevated otherwise
│  ← (back) | Title (serif h2)    │  ← back arrow en sub-screens; root screens muestran avatar right
├─────────────────────────────────┤
│                                 │
│                                 │
│      Main content area          │  ← scroll, full bleed
│      max-width 480px (mobile)   │
│      max-width 840px (desktop)  │
│                                 │
│                                 │
├─────────────────────────────────┤
│ 📅 🗓 ☰ 🎯 💬 📁 ⚙️             │  ← Bottom nav 64px height, 7 items
│ Today Plan Tasks Goals Chat     │     Categorías Settings
└─────────────────────────────────┘
            🎙️                     ← FAB mic, 56px, bottom-right (above bottom nav)
```

**Bottom nav items (orden):**

1. **Today** (📅) — SCR-020
2. **Plan** (🗓) — SCR-021 (default Week tab; submenú Week/Month — ver `WeekMonthTabs`)
3. **Tasks** (☰) — SCR-025
4. **Goals** (🎯) — SCR-022
5. **Chat** (💬) — SCR-023
6. **Categorías** (📁) — SCR-026
7. **Settings** (⚙️) — SCR-024

Each: icon (20-24px Lucide) + label (caption, 11px). Active state: ink-primary; inactive: ink-hint. NO color accents in nav itself.

**Overflow rule:** si el viewport es < 360px y los 7 items no caben, los últimos 2 (Categorías + Settings) colapsan en un menú "Más" (icono ⋯). En tablet/desktop, los 7 items siempre caben.

**FAB mic:**

- Visible en Today, Plan, Tasks, Goals, Categorías (NO en Chat — chat tiene su propio mic en input bar)
- Position: bottom-right, `bottom: calc(--bottom-nav-height + 16px); right: 16px`
- Z-index above content, below modals
- Tap → SCR-050 Voice capture sheet

**Safe areas:** bottom nav respects `safe-area-inset-bottom`. FAB stays clear of home indicator.

### Tablet (768-1023px) y Desktop (≥ 1024px)

Misma estructura que mobile: bottom nav horizontal abajo. Contenido centrado max-width 840px con side padding. **NO existe sidebar.** Esta decisión simplifica responsive y es coherente con producto mobile-first multi-tenant single-user-data.

## §5 — Form Specifications

### Pattern principles

1. **Sin floating labels.** Label siempre arriba, caption uppercase. Floating labels son anti-pattern editorial (Designs that survive: Material).
2. **Errors inline below field.** Color `--danger`, text-small. No modal de errores.
3. **Validation onBlur + onSubmit.** No onChange (intrusivo).
4. **Submit con Enter en single-field forms.** Activity quick-add, search, etc.
5. **Disabled state:** ink-hint sobre bg-sunken. Opacity NOT used (lo hace muerto, no clearly disabled).
6. **Required marker:** `*` after label en ink-soft. No "required" word.
7. **Optional marker:** `(opcional)` after label en ink-hint, body-s.
8. **Help text:** caption italic serif below field cuando aplica.

### Form patterns

#### F-1 — Activity quick-add (inline en Today, sin modal)

```
[ Title input (autofocus, placeholder italic serif) ]
[ Project dropdown ↓ Inbox ]   [ Date: Hoy ↓ ]   [ Prio: ◯◯◯●● ]
[ + más detalles ▼ ]                              [ Crear → ]
```

Expandido:

```
[ Description (textarea, 2 rows autosize) ]
[ Time (opcional) ]   [ Deadline (opcional) ]
[ Estimated min ]     [ Recurrence ]
[ Tags chips input ]
[ Vincular goal ↓ ]
```

#### F-2 — Activity full form (en detail screen)

Mismos campos pero stacked vertical, con sections agrupadas (Title+Description / Schedule / Priority+Tags / Linkage).

#### F-3 — Sheet field inline edit

Pattern: tap field → field becomes editable textarea → blur or Cmd+Enter to save → optimistic UI update + Toast "Guardado."

Placeholder es **italic serif evocativo**:

- intention: _"Una intención, en una frase"_
- gratitude: _"Algo por lo que estás agradecido"_
- identity: _"Hoy soy alguien que..."_
- evening*lesson: *"Lo que aprendí hoy"\_

#### F-4 — Goal create / edit

```
[ Title * ]
[ Description (opcional) ]
[ Scope * (Quarter / Year / 5-Year / Life) - radio cards ]
[ Deadline * ] (required for quarter/year, optional for 5year/life)
[ Outcome expected (opcional) ]   ← help: caption italic "Cómo se ve cuando se cumple"
[ Cancelar ] [ Crear goal → ]
```

#### F-5 — Goal review (modal SCR-053)

```
[ Calificación: 1 ──●── 10 ]   (slider con marks 1, 5, 10)
[ Notes (textarea opcional) ]   ← help: "Qué cumpliste, qué no, por qué"
[ Status sugerido: ●Achieved ◯Partial ◯Abandoned ]   ← derived from score (BR-9), user can override
[ Cancelar ]   [ Guardar review ]
```

#### F-6 — Notification preferences

```
SECCIÓN: HORARIOS

[ Morning check-in     08:00 ↓ ]
[ Midday check-in      13:00 ↓ ]
[ Evening check-in     21:00 ↓ ]

──────────────────────────────────

SECCIÓN: SEMANA
```

## §7 — Design Decisions (DD-XXX)

| ID          | Decision                                                                      | Rationale                                                                                                                                                                                                                             | Trade-offs                                                                                                                                             |
| ----------- | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **DD-001**  | Bottom nav 5 items en mobile (Today / Week / Goals / Chat / Settings)         | 5 es el límite de discriminación visual en touch UI (Nielsen). Cubre las navegaciones primarias del producto.                                                                                                                         | Tablet con 768px+ podría ganar con sidebar siempre visible — pero la mayoría de tablet users usan portrait, bottom nav sigue OK.                       |
| **DD-002**  | FAB mic en lugar de mic en chat                                               | Captura debe ser accesible desde Today/Plan/Tasks/Goals/Categorías sin abrir Chat. FAB es el patrón mobile estándar para acción primaria contextual. (Actualizado por iteración prototipo: ahora visible en 5 screens en lugar de 3.) | Chat tiene su propio mic en input bar — coexisten.                                                                                                     |
| **DD-003**  | Warm-book aesthetic en lugar de iOS-native glassmorphism                      | Diferenciador real vs competidores. Coherente con tono "asistente reflexivo, no app productivity-coach".                                                                                                                              | Algunos users tech-savvy podrían leerlo como "old-fashioned". Mitigación: warm pero contemporáneo (Source Serif 4, no Garamond).                       |
| **DD-004**  | Light mode como canonical (dark optional toggle)                              | Paleta warm-book nace de cream + warm charcoal. Light es el "estado canónico" del libro. Dark mode existe (modo nocturno) pero NO inversión simple — paleta calibrada.                                                                | Conflicto con P-1 tech-savvy que prefiere dark default. Mitigación: toggle visible en settings. Telemetría: si >40% usan dark, evaluar default switch. |
| **DD-005**  | Sheets como "páginas" con scope accent bar lateral                            | Refuerza metáfora cuaderno. | **DD-011**  | Sin floating labels en inputs                                                 | Anti-pattern editorial; reduce usability cuando label es vital info.                                                                                                                                                                  | Más espacio vertical necesario por field.                                                                                                              |
| **DD-012**  | Toast top-center mobile (no bottom)                                           | No conflicta con bottom nav + FAB. ## Multi-perspective review

### 1. Product perspective

- ✅ North Star "accountability activa" se materializa en check-ins (FLW-004..007), vague-answer challenges (FLW-013), and post-mortem (DD-005)
- ✅ Captura frictionless en FAB mic + 1-tap confirm (FLW-003, DD-002, DD-008)
- ⚠️ Risk: 41 screens es ambicioso para 10 semanas — backlog tendrá que priorizar P0/P1 strict

### 2. Architecture perspective

- ✅ Routing map alineado con Next.js App Router conventions
- ✅ Server Actions cubren mutations; Route Handlers solo streaming/upload/webhooks
- ✅ Deep links from push notifications preservan context (system prompt selection en chat)

### 3. Frontend perspective

- ✅ TimeKast kit cubre primitives (shadcn/ui base); ~50 new CMP a buildear
- ✅ Bottom-sheet pattern para voice capture es estándar (vaul library)
- ⚠️ Drag-to-reorder en categories requiere dnd-kit (no native HTML5)
- ✅ OQ-D-9 (2-column sheet desktop) — **resuelto**: nav SIEMPRE horizontal (DD-031), desktop usa el mismo layout que mobile. No hay sidebar que liberar espacio para una segunda columna.

### 4. UX perspective

- ✅ One question per turn del agente preservado en FLW-004..007
- ✅ Crisis exit es UI takeover (DD-015) — safety priority
- ✅ Italic serif placeholders evocativos refuerzan voice (DD-023)
- ⚠️ Activity swipe direction (OQ-D-8) — preference testing necesario
- ⚠️ Chat sin bubble convencional (DD-006) es non-standard — onboarding debe modelar

### 5. Accessibility perspective

- ✅ Touch targets ≥48px en todos los wireframes
- ✅ Contrast ratios validable con tokens definidos
- ⚠️ Color-blind users: per-scope accents need text labels alongside (AS-D-2)
- ⚠️ Italic placeholders contrast (AS-D-7) needs validation tool
- ✅ Screen reader: SCR labels, ARIA on icon buttons, keyboard nav en desktop
- ⚠️ Voice capture flow needs visible captions (no solo audio)
